🧭 開發人員交付說明

1. 畫面邏輯與導航

  • 此應用程式圍繞標準的 SwiftUI `TabView` 構建，現在有 4 個分頁。
  • 首頁 (`#home-screen`): 應用程式的入口點。對應 `HomeView.swift`。
  • 占卜結果 (`#reading-result-screen` & `#reading-result-saved-screen`): 顯示抽出的牌卡。`#reading-result-saved-screen` 展示了點擊儲存按鈕後的 UI 狀態。開發時，點擊「儲存至日誌」按鈕應：
    1. 使用 `Date()` 獲取當前時間戳。
    2. 將結果（牌陣、牌卡、時間戳）使用 `SwiftData` (或 Core Data) 保存。
    3. 更新 UI 至 "已儲存" 狀態，如禁用按鈕並顯示時間。
  • 牌卡圖書館 (`#library-screen`): 第二個分頁，對應 `CardLibraryView.swift`。
  • 占卜日誌 (`#journal-screen`): 新增的第四個分頁。從 `SwiftData` 讀取並顯示紀錄列表。對應 `JournalView.swift`。
  • 設定 (`#settings-screen`): 第三個分頁。包含用於切換外觀模式和推播通知的 `Toggle`。對應 `SettingsView.swift`。

2. SwiftUI 元件映射

  • 手機框架: 這是一個視覺指南。SwiftUI 中的主要容器將是 `ContentView.swift`，包含 `TabView`。
  • 塔羅牌卡 (`.tarot-card`): 可重用的 `TarotCardView.swift`。佔位圖示應替換為實際的牌卡藝術作品。
  • 列表: 設定和日誌畫面使用 SwiftUI `List` 與 `Section` 標題。
  • 切換開關 (`.toggle-switch`): 對應到 SwiftUI 的 `Toggle` 元件，並使用 `.tint()` 修改器來設定開啟時的顏色。

3. 資源與樣式

  • 調色板:
    • 背景: `slate-950` (#020617)
    • 主要文本: `slate-200` (#E2E8F0)
    • 次要文本: `slate-400` (#94A3B8)
    • 主要強調色: `purple-500` (#A855F7)
    • 次要強調色: `amber-400` (#F59E0B)
    這些顏色應作為顏色集添加到 `Assets.xcassets` 文件中，以便在 SwiftUI 中輕鬆使用。
  • 字體: 原型使用系統字體堆疊。在 SwiftUI 中，請使用 `.font(.largeTitle)`、`.font(.headline)` 等，以符合 HIG 規範。
  • 圖示: 圖示來自 Heroicons。可以導出為 SVG 並添加到 `Assets.xcassets` 中，或安裝為 SF Symbols。
  • 牌卡藝術作品: 佔位符牌卡內容 (SVG) 必須替換為 78 張獨特的塔羅牌卡圖像。
